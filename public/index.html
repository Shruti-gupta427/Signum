1

Send email notification

POST

/notify

Header: Content-Type: application/json

Request body 
{
  "userId": "user_1",
  "channel": "email",
  "email": "your@gmail.com",
  "message": {
    "subject": "Signum Test",
    "body": "Signal everything, miss nothing."
  },
  "priority": 0
}
Expected response 
{
  "queued": true,
  "id": "xxxxxxxxxxxxxxxx"
}

✓ Email received in inbox

  "priority": 0

4

SSE connection

GET

/subscribe/sse?userId=user_1

Open in browser (keeps connection open):

http://localhost:3000/subscribe/sse?userId=user_1

✓ Browser stays connected · ✓ Terminal shows connection added

Then send this via POST /notify:

Notify body (SSE channel) 
{
  "userId": "user_1",
  "channel": "sse",
  "message": {
    "title": "Live!",
    "body": "SSE works!"
  },
  "priority": 1
}

✓ Browser receives notification instantly

API reference

Endpoints

MethodEndpointDescription
POST/notifySend single notification
POST/notify/broadcastSend to multiple users
GET/subscribe/sse?userId=xxxSSE connection
GET/notifications/:userIdNotification history
GET/notifications/:userId/unreadUnread count
PATCH/notifications/:id/readMark as read

Status

StatusMeaning
queuedAdded to queue
processingBeing delivered
deliveredSuccessfully sent
pendingUser offline
failedAll retries exhausted
rate_limitedToo many requests
readUser read it

Priority

LevelValueUse case
CRITICAL0OTP, security alerts
HIGH1Payments, orders
MEDIUM2Reminders
LOW3Promotions

Channels

ChannelDescription
emailEmail via Nodemailer
smsSMS stub (console.log)

sseServer-Sent Events
